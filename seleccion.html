Método de ordenación de Selección

Este método de ordenamiento se caracteriza por realizar comparaciones de cada uno de los valores con todos los demás de manera secuencial, con el objetivo de seleccionar el valor menor para ubicarlo en un orden nuevo, es decir se compara el 1 con el 2, el 1 con el 3 y así sucesivamente, si en una comparación se encuentra que el valor comparado con el primer dato es menor este pasa a ser el posible menor valor del arreglo y se guarda en una variables auxiliar, si al encontrar en una nueva comparación un nuevo valor menor que el primero, este nuevo valor es comparado con la variable guardada para decidir si pasa a ser el nuevo valor menor, al final del primer recorrido se ubica en una nueva lista el menor valor, esta lista se va llenando de acuerdo a los recorridos realizados para obtener el ordenamiento deseado.




Este tipo de ordenamiento maneja unicamente un espacio de memoria adicional donde se almacena el valor a comparar, por lo cual su requerimiento en memoria es constante y no requiere de memoria extra, tambien se considera de facil utilizacion y funcionalidad constante ya que son inperceptibles las diferencias entre una comparacion y la subsiguiente, algunos aspectos que ponen en desventaja de este metodo con respecto a otros es que se torna lento ademas de realizar muchas comparaciones, se recomienda el uso para manejo de bases con registros grandes y claves cortas.



A continuación, se muestra el algoritmo para su debida implementación.


Entero[] seleccion(entero v[], entero n)
	Entero: i, j, posmenor, aux
		Para i = 1 hasta n-1 hacer
			posmenor = i
			Para j = i+1 hasta n hacer
				Si v[posmenor] > v[j] entonces
					posmenor = j
				Fin si
			Fin para
			aux = v[i]
			v[i] = v[posmenor]
			v[posmenor] = aux
		Fin para
	Retornar v[]
Fin selección
